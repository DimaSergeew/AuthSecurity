# AuthSecurity — Настройка веб-капчи (Cloudflare Turnstile)

## Что это

AuthSecurity показывает новым игрокам одноразовую веб-капчу перед регистрацией — это защищает сервер от ботов.
Используется Cloudflare Turnstile (бесплатно). Капча выключена по умолчанию.

---

## Шаг 1 — Получить ключи Cloudflare Turnstile

1. Зайди на https://dash.cloudflare.com → раздел **Turnstile**
2. Нажми **Add site**
3. Тип виджета — **Managed** (рекомендуется)
4. В поле Domains укажи домен страницы капчи (например `captcha.example.com`)
5. Скопируй **Site Key** (публичный) и **Secret Key** (приватный)

> Тестовые ключи (уже прописаны в config.yml) делают капчу всегда успешной —
> они подходят для локальной проверки, но ЗАМЕНИ их на реальные перед продакшном.

---

## Шаг 2 — Настроить config.yml

Открой `plugins/AuthSecurity/config.yml`, найди секцию `captcha:` и измени:

```yaml
captcha:
  enabled: true
  site-key: "ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ"
  secret-key: "ВАШ_ПРИВАТНЫЙ_КЛЮЧ"
  web-bind: "127.0.0.1"            # 127.0.0.1 если используешь прокси, иначе 0.0.0.0
  web-port: 25590                  # порт встроенного HTTP-сервера
  public-base-url: "https://captcha.example.com"   # URL который увидит игрок
```

> **Важно:** параметры `web-bind`, `web-port` и секция `database:` применяются
> только после полного перезапуска сервера. Остальное — командой `/authsecurity reload`.

---

## Шаг 3 — Схема развёртывания

### Вариант А — Быстрый тест (без прокси)

Подходит для проверки работы на локальной машине. IP сервера будет виден игрокам.

```yaml
web-bind: "0.0.0.0"
public-base-url: "http://ВАШ_IP:25590"
```

Открой порт 25590 в firewall. После перезапуска сервера страница капчи должна
открываться по адресу `http://ВАШ_IP:25590/c/test`.

---

### Вариант Б — Продакшн через Gcore CDN (рекомендуется)

Gcore CDN скрывает реальный IP сервера и доступен из России, Украины и всей Европы
без региональных ограничений. Тарифный план Free — бесплатно.

**1.** Зарегистрируйся на **gcore.com**

**2.** Перейди в раздел CDN → **Создать CDN-ресурс**
- Домен (CNAME): `captcha.ваш-домен.com`
- Источник: `ВАШ_IP` с портом `25590`
- DNS: **Не делегировать**

**3.** Добавь CNAME-запись в DNS своего домена (например в Cloudflare):
```
Тип:   CNAME
Name:  captcha
Value: cl-xxxxxxxx.gcdn.co   ← адрес из панели Gcore
Proxy: DNS only (серое облако, НЕ проксировать через Cloudflare)
```

**4.** Настрой config.yml:
```yaml
captcha:
  web-bind: "127.0.0.1"
  web-port: 25590
  public-base-url: "http://captcha.ваш-домен.com"
```

**5.** Закрой порт 25590 для внешних подключений (принимать трафик только от Gcore):
```bash
# UFW (Ubuntu/Debian):
ufw deny 25590
ufw allow from IP_GCORE to any port 25590
```

---

### Вариант В — Продакшн через Nginx на своём VPS

Подходит если у тебя уже есть VPS. Nginx скрывает реальный IP.

**1.** Установи Nginx:
```bash
apt install nginx -y
```

**2.** Создай конфиг `/etc/nginx/sites-available/captcha`:
```nginx
server {
    listen 80;
    server_name captcha.ваш-домен.com;

    location / {
        proxy_pass http://IP_ИГРОВОГО_СЕРВЕРА:25590;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
ln -s /etc/nginx/sites-available/captcha /etc/nginx/sites-enabled/
nginx -t && nginx -s reload
```

**3.** Настрой DNS: A-запись `captcha.ваш-домен.com` → IP VPS (без Cloudflare прокси)

**4.** Настрой config.yml:
```yaml
captcha:
  web-bind: "0.0.0.0"
  web-port: 25590
  public-base-url: "http://captcha.ваш-домен.com"
```

**5.** Закрой порт 25590 на игровом сервере для всех кроме IP VPS:
```bash
ufw deny 25590
ufw allow from IP_VPS to any port 25590
```

## Справочник параметров секции captcha

| Параметр                       | По умолчанию            | Описание                                                          |
|--------------------------------|-------------------------|-------------------------------------------------------------------|
| `enabled`                      | `false`                 | Включить капчу                                                    |
| `site-key`                     | тестовый ключ           | Публичный ключ Cloudflare Turnstile (вставляется в HTML)          |
| `secret-key`                   | тестовый ключ           | Приватный ключ (только на сервере, никогда не светить публично)   |
| `web-bind`                     | `0.0.0.0`               | IP интерфейса для встроенного HTTP-сервера                        |
| `web-port`                     | `25590`                 | Порт HTTP-сервера ⚠ требует перезапуска сервера                   |
| `public-base-url`              | `http://127.0.0.1:25590`| URL страницы капчи который откроется в браузере игрока            |
| `token-ttl-minutes`            | `10`                    | Как долго действует ссылка капчи (мин)                            |
| `verification-validity-days`   | `7`                     | Через сколько дней просить капчу снова (0 = каждый раз)           |
| `refresh-verification-on-login`| `false`                 | Обновлять срок капчи при каждом успешном входе                    |
| `revalidate-on-ip-change`      | `true`                  | Требовать повторную капчу при смене IP (отключи для мобильных)    |
| `max-concurrent-challenges`    | `50`                    | Максимум одновременных проверок (защита от перегрузки потоков)    |

## Устранение неполадок

**Сервер не запускается, ошибка в консоли про captcha web server:**
- Порт 25590 уже занят другим процессом — измени `web-port` и перезапусти
- Неверный `web-bind` — попробуй `0.0.0.0`

**Страница капчи не открывается у игрока:**
- Проверь что `public-base-url` доступен из браузера: открой `<public-base-url>/c/test`
- Убедись что порт 25590 открыт в firewall (или работает прокси)

**Виджет Turnstile не загружается:**
- Убедись что `site-key` — публичный ключ (начинается с `1x` у тестового или с `0x`)
- Включён ли JavaScript в браузере игрока?

**Cloudflare не подтверждает капчу (игрок проходит, но вход не продолжается):**
- Проверь `secret-key` — он должен быть приватным (не публичным)
- Игровой сервер должен иметь исходящий HTTPS-доступ к `challenges.cloudflare.com`

**Игроки из России или Украины не могут открыть страницу:**
- Используй **Вариант Б** (Gcore CDN) — он доступен из обеих стран без ограничений
- Gcore бесплатен для базового трафика
